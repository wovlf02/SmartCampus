# 용어 정의

**관련 문서**: [프로젝트 개요](./project-overview.md)

---

## 1. 일반 용어

| 용어 | 설명 |
|------|------|
| **Smart Campus** | 본 프로젝트 명칭. 교내 네비게이션 및 커뮤니티 앱 |
| **사용자 (User)** | 앱을 사용하는 모든 회원 |
| **대학교 (University)** | 사용자가 소속된 대학교 |

---

## 2. 인증 관련 용어

| 용어 | 설명 |
|------|------|
| **JWT (JSON Web Token)** | 사용자 인증에 사용되는 토큰 형식 |
| **Access Token** | API 접근 권한을 위한 단기 토큰 (유효시간 짧음) |
| **Refresh Token** | Access Token 갱신을 위한 장기 토큰 |
| **소셜 로그인** | 카카오, 네이버, 구글, GitHub 등 외부 인증 |
| **이메일 인증** | 회원가입 시 이메일로 인증코드 발송 및 검증 |

---

## 3. 커뮤니티 관련 용어

| 용어 | 설명 |
|------|------|
| **게시글 (Post)** | 커뮤니티 게시판에 작성되는 글 |
| **댓글 (Comment)** | 게시글에 달리는 1차 응답 |
| **대댓글 (Reply)** | 댓글에 달리는 2차 응답 |
| **좋아요 (Like)** | 게시글에 대한 긍정적 반응 표시 |
| **첨부파일 (Attachment)** | 게시글에 첨부된 이미지/파일 |
| **신고 (Report)** | 부적절한 콘텐츠 또는 사용자 신고 |
| **차단 (Block)** | 특정 사용자의 콘텐츠 숨김 처리 |
| **즐겨찾기 (Favorite)** | 게시글 즐겨찾기 |

---

## 4. 채팅 관련 용어

| 용어 | 설명 |
|------|------|
| **채팅방 (ChatRoom)** | 메시지를 주고받는 대화 공간 |
| **1:1 채팅 (Direct)** | 두 사용자 간의 개인 채팅 |
| **그룹 채팅 (Group)** | 3인 이상 사용자의 그룹 채팅 |
| **메시지 (ChatMessage)** | 채팅방에서 주고받는 개별 메시지 |
| **읽음 표시 (Read)** | 메시지를 읽었는지 여부 표시 |
| **참여자 (Participant)** | 채팅방에 참여 중인 사용자 |
| **WebSocket** | 실시간 양방향 통신 프로토콜 |
| **STOMP** | WebSocket 상에서 메시지 라우팅을 위한 프로토콜 |
| **SockJS** | WebSocket 폴백을 제공하는 라이브러리 |

---

## 5. 친구 관련 용어

| 용어 | 설명 |
|------|------|
| **친구 (Friend)** | 서로 친구 관계가 수립된 사용자 |
| **친구 요청 (FriendRequest)** | 친구 관계 수립을 위한 요청 |
| **친구 차단 (FriendBlock)** | 특정 친구를 차단 |
| **친구 신고 (FriendReport)** | 친구 관계의 사용자를 신고 |

---

## 6. 시간표 관련 용어

| 용어 | 설명 |
|------|------|
| **시간표 (Timetable)** | 사용자의 수업/일정 스케줄 |
| **요일 (DayOfWeek)** | 수업이 있는 요일 (월~일) |
| **교시** | 수업 시간 단위 |

---

## 7. 기술 용어

| 용어 | 설명 |
|------|------|
| **React Native** | 크로스 플랫폼 모바일 앱 개발 프레임워크 |
| **Spring Boot** | Java 기반 백엔드 프레임워크 |
| **JPA (Java Persistence API)** | Java ORM 표준 인터페이스 |
| **Entity** | 데이터베이스 테이블과 매핑되는 Java 클래스 |
| **DTO (Data Transfer Object)** | 계층 간 데이터 전송 객체 |
| **Repository** | 데이터 접근 계층 인터페이스 |
| **Service** | 비즈니스 로직 처리 계층 |
| **Controller** | HTTP 요청 처리 계층 |
| **REST API** | HTTP 기반 자원 중심 API 설계 방식 |
| **i18n (Internationalization)** | 다국어 지원 |
| **T Map SDK** | SK텔레콤 티맵 지도 서비스 SDK |

---

## 8. 상태 값 정의

### 8.1 친구 요청 상태 (FriendRequestStatus)

| 값 | 설명 |
|----|------|
| PENDING | 대기 중 |
| ACCEPTED | 수락됨 |
| REJECTED | 거절됨 |

### 8.2 신고 상태 (ReportStatus)

| 값 | 설명 |
|----|------|
| PENDING | 처리 대기 |
| REVIEWED | 검토 완료 |
| RESOLVED | 처리 완료 |

### 8.3 채팅방 유형 (ChatRoomType)

| 값 | 설명 |
|----|------|
| DIRECT | 1:1 채팅 |
| GROUP | 그룹 채팅 |
| STUDY | 스터디 그룹 채팅 |
| POST | 게시글 연동 채팅 |

### 8.4 메시지 유형 (ChatMessageType)

| 값 | 설명 |
|----|------|
| TEXT | 텍스트 메시지 |
| IMAGE | 이미지 메시지 |
| FILE | 파일 메시지 |
| ENTER | 입장 알림 |
| LEAVE | 퇴장 알림 |
